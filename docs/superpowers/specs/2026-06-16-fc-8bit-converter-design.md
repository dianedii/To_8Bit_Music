# FC/NES 8-bit 芯片音乐转换器设计文档

**日期：** 2026-06-16  
**主题：** 钢琴 MP3 一键转纯正 FC 8-bit 芯片音乐  
**状态：** 待实现

---

## 1. 目标与范围

开发一款基于 PyQt6 的桌面 GUI 工具，将输入的纯钢琴 MP3 一键转换为纯正红白机（FC/NES）风格的 8-bit 芯片音乐。

**核心要求：**
- 仅保留单声部主旋律，剔除伴奏、和弦、低音声部。
- 不使用原音频滤镜或频谱改造，必须走「分离 → 识别 → 重合成」全链路，从零生成纯电子音频。
- 默认输出 MP3，可选 WAV。
- 单首时长 ≤ 10 分钟，3 分钟以内曲目转换耗时 ≤ 180 秒。
- 本地安装依赖后可一键启动，依赖优先通过国内镜像安装。

---

## 2. 技术路线

采用「分离 - 识别 - 重合成」全链路：

```
MP3 钢琴音频
    ↓
[piano_transcription_inference]  钢琴专用转录模型
    ↓
MIDI 音符事件（音高 / 起止时间 / 力度）
    ↓
[主旋律提取]  时序聚类 + 主导声部打分
    ↓
单声部主旋律序列
    ↓
[音符简化]  碎音过滤 / 同音合并 / 颤音装饰音移除 / 密度限制
    ↓
干净的单声部音符序列
    ↓
[FC 方波合成]  从零生成方波波形
    ↓
MP3 / WAV 输出
```

全程不保留任何原始音频采样或音色特征。

---

## 3. 功能需求

### 3.1 输入
- 支持格式：MP3
- 时长限制：≤ 10 分钟，超出时弹出友好提示并终止转换

### 3.2 GUI（PyQt6）
- 极简图形界面，无需命令行
- 核心控件：
  - 选择文件按钮 + 文件路径显示
  - 开始转换按钮
  - 进度条
  - 运行状态提示
  - 转换完成后「打开文件所在文件夹」按钮
- 可调参数：
  - **复古纯度**（滑块）：最左为纯 FC 方波，最右为轻微润色（vibrato + 柔和 release）
  - **音符简化强度**（滑块）：过滤碎音 / 装饰音 / 颤音，强度越高旋律越干净
  - **整体音量**（滑块）：控制输出音量，防止破音
  - **输出格式**（下拉框）：MP3（默认）/ WAV

### 3.3 输出
- 转换完成自动生成结果文件
- 默认 MP3（192kbps CBR），可选 WAV（16bit / 44.1kHz）
- 提供「打开文件所在文件夹」按钮

## 4. 架构与模块

| 文件 | 职责 |
|------|------|
| `main.py` | 程序入口，初始化 GUI |
| `ui.py` | PyQt6 主窗口、控件布局、信号绑定、文件对话框 |
| `worker.py` | 后台转换线程（QThread），避免界面卡死，emit 进度和状态 |
| `transcriber.py` | 封装 `piano_transcription_inference`，MP3 → 音符事件列表 |
| `melody_extractor.py` | 时序聚类提取主旋律声部，剔除伴奏/和弦/低音 |
| `note_simplifier.py` | 碎音过滤、同音合并、颤音/装饰音移除、密度限制 |
| `synthesizer.py` | 纯 FC 方波合成，支持 vibrato、release、防破音限幅 |
| `utils.py` | 依赖检测、国内镜像 pip 安装辅助、音频导出 |

---

## 5. 数据流

1. 用户选择 MP3 文件，点击「开始转换」。
2. Worker 线程读取音频，检查时长是否超过 10 分钟。
3. 调用 `piano_transcription_inference` 将钢琴音频转为 MIDI 级音符事件。
4. `melody_extractor.py` 对音符做时间窗聚类，在每个时间窗内按「时长 × 力度权重 + 音高奖励」打分，选出主旋律音。
5. `note_simplifier.py` 根据「音符简化强度」参数过滤碎音、合并相邻同音、移除颤音/装饰音、限制每拍最大音符密度。
6. `synthesizer.py` 根据「复古纯度」参数生成 FC 方波，加入轻微 vibrato 与 release 尾音，最终做软限幅防止破音。
7. `utils.py` 按所选格式导出为 MP3 或 WAV。
8. GUI 显示完成状态，启用「打开文件所在文件夹」按钮。

---

## 6. 参数映射

| 参数 | 控件 | 范围 | 作用 |
|------|------|------|------|
| 复古纯度 | 滑块 | 0 ~ 100，默认 0 | 0=纯 50% 占空比方波；100=加入轻微 vibrato 和柔和 release 尾音 |
| 音符简化强度 | 滑块 | 0 ~ 100，默认 50 | 控制最小音符时长、同音合并窗、颤音移除窗、每拍最大音符数 |
| 整体音量 | 滑块 | 0 ~ 100，默认 80 | 最终输出 gain，防止破音 |
| 输出格式 | 下拉框 | MP3（默认）/ WAV | MP3 使用 192kbps CBR；WAV 为 16bit / 44.1kHz 无损 |

## 7. 关键算法概述

### 7.1 主旋律提取
- 将转录结果整理为 `(pitch, onset, offset, velocity)` 事件列表。
- 以 50ms 为窗口扫描，将重叠或紧邻的音符分组为「同时响起的音符群」。
- 在每个群里按「音符时长 × 力度权重 + 音高奖励」打分，最高分作为该时刻的主旋律音。
- 最终得到一条单声部主旋律序列。

### 7.2 音符简化
- **碎音过滤：** 丢弃时长短于阈值的孤立音符。
- **同音合并：** 相邻同音高且间隔小于阈值的音符合并为一个持续音符。
- **颤音/装饰音移除：** 检测极短时间内反复上下跳的小音符群，只保留第一个或最长的一个。
- **密度限制：** 每拍最多保留 N 个音符，优先保留更长、更响的音符。

### 7.3 FC 方波合成
- 采样率 44.1kHz，单声道。
- 基础波形为 50% 占空比方波。
- 根据「复古纯度」加入轻微 vibrato（频率调制）和 release 尾音。
- 音符切换处做极短 ramp，避免爆音。
- 最终做软限幅（soft clipping）和归一化，确保无破音。

## 8. 依赖管理

主要依赖：
- `PyQt6` — GUI
- `piano_transcription_inference` — 钢琴音频转 MIDI
- `numpy` / `scipy` — 数值计算与音频合成
- `pydub` 或 `lameenc` — MP3 导出

安装策略：
- 程序启动时检测关键依赖是否已安装。
- 缺失依赖时，优先使用国内 PyPI 镜像（如清华大学 / 阿里云镜像）自动安装。
- 提供一键安装脚本，避免用户手动配置 pip。

## 9. 验收标准

### 9.1 音质
- 零原钢琴音色残留，无泛音、共鸣。
- 无杂音、底噪、爆音、破音；高音不刺耳，低音不发糊。
- 音符边界清晰，无粘连、无碎音堆砌。
- 音高准确，熟悉曲目可直观识别。

### 9.2 风格
- 极致纯正 FC 芯片质感，标准方波主音色。
- 无混响、回声、立体声特效。
- 仅保留单声部主旋律，不添加伴奏/和声/低音。
- 复杂快速段落自动简化，不嘈杂。

### 9.3 功能
- GUI 可正常启动、选文件、转换，无崩溃、无卡死。
- 3 分钟以内曲目转换耗时 ≤ 180 秒。
- 参数调节对下一次转换生效，效果差异可感知。

## 10. 风险与应对

| 风险 | 应对 |
|------|------|
| `piano_transcription_inference` 模型首次下载大、慢 | 提供国内镜像/预下载脚本，首次下载后缓存复用 |
| CPU 推理速度慢，3 分钟曲目可能接近 180 秒上限 | 采用后台线程 + 进度条；必要时可提示用户等待 |
| 复杂钢琴曲主旋律提取不准 | 通过「音符简化强度」参数给用户调节空间；算法预留调参接口 |
| MP3 导出依赖外部编码器 | 优先使用 `lameenc` 或 `pydub`，缺失时 fallback 到 WAV |
